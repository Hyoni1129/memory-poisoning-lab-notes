# Logging Workflow

This workflow helps keep notes useful and lightweight.

## 1. During Work
- keep a running `daily-YYYY-MM-DD.md` note
- add key actions and why they were taken
- mark uncertainty explicitly

## 2. At Decision Points
- create or update a decision note immediately
- include alternatives and risks

## 3. After Experiments
- add a process-focused note in `05_experiment_notes/`
- include the experiment ID and links to formal outputs

## 4. End Of Session
- summarize what was learned
- list next actions
- mark blockers and unresolved questions

## 5. Weekly Maintenance
- update reading index statuses
- connect daily notes to decision and experiment notes
