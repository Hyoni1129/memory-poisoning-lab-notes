# Setup Summary

This repository was scaffolded as the process and lab-notebook companion to the formal research repository.

## Repository Purpose
- Preserve the research journey with high transparency.
- Capture reasoning that does not belong in the formal artifact repository.

## What Was Scaffolded
- Root onboarding files:
  - `README.md`
  - `SETUP_SUMMARY.md`
  - `GITHUB_CREATION_INSTRUCTIONS.md`
  - `COMMIT_HELPER.md`
- Process directories (`00_` to `09_`) with local guidance README files.
- Reusable templates for all major note types in `templates/`.
- Documentation guides in `docs/` for naming, logging, commit linking, and public policy.

## Established Conventions
- Date format in filenames: `YYYY-MM-DD`
- Shared experiment IDs with `memory-poisoning-research`
- Prefix-based commit messages with lightweight cross-references

## What To Do First
1. Read `docs/how-to-use-this-repo.md`.
2. Create your first note in `03_daily_log/` using the daily template.
3. Add a living reading index entry in `08_reading_list/reading-index.md`.
4. Add decisions immediately in `02_decision_log/`.
