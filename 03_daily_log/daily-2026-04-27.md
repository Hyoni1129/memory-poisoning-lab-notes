# Daily Log: 2026-04-27

## 상세 학습 내용 (Details)
오늘은 'LLM 애플리케이션을 위한 OWASP Top 10 2025' 문서를 읽고 LLM 환경에서 발생할 수 있는 주요 보안 위협과 취약점에 대해 학습했습니다. 특히 다음의 주요 항목들에 대해 이해도를 높였습니다.

1. **프롬프트 인젝션 (Prompt Injection)**
   - 악의적인 프롬프트 입력을 통해 LLM의 원래 지침(System Prompt)을 우회하거나 개발자가 의도하지 않은 행동을 유발하는 공격.
2. **데이터 및 모델 오염 (Data and Model Poisoning)**
   - LLM의 학습 단계나 파인튜닝, RAG 등에 사용되는 데이터에 악의적인 정보를 주입하여 모델의 출력 결과를 조작하거나 보안을 훼손하는 취약점. (현재 진행 중인 메모리 포이즈닝 연구와 매우 밀접한 관련이 있음)
3. **공급망 보안 (Supply Chain Security)**
   - 외부 라이브러리, 서드파티 데이터셋, 사전 학습된 모델, 플러그인 등 LLM 서비스 구성 요소에서 발생하는 취약점을 통한 공격 위험.
4. **시스템 프롬프트 유출 (System Prompt Leakage)**
   - 사용자와의 상호작용 과정에서 모델을 보호하거나 제어하기 위해 설정된 기밀 시스템 프롬프트가 노출되는 현상.

## 생각 및 향후 계획 (Notes & Next Steps)
- OWASP Top 10에서 언급된 취약점 중 **데이터/모델 오염(Poisoning)**과 **프롬프트 인젝션(Prompt Injection)**이 에이전트의 장기 기억(Long-term Memory)에 미치는 영향에 대해 추가적인 연구 아이디어를 구체화할 필요가 있습니다.
